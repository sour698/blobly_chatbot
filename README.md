# Blobly Assistant

An AI-powered chatbot that helps users understand **Blobly** — its
company overview, mission, vision, products (Nova, WASP, Forge), core
values, internship program, offer-letter verification, and contact
information. Answers are generated by the **Google Gemini API**, grounded
in a Blobly-specific knowledge base so responses stay accurate and on-brand.

---

## Project Overview

Blobly Assistant is a full-stack chat application built for Blobly's
website. Visitors can type a question — or tap one of the suggested
prompts (e.g. *"What is Nova?"*, *"How can I apply for a Blobly
internship?"*) — and get an instant, on-brand answer.

The app is split into two parts:

- A **frontend** chat interface (HTML/CSS/JS) styled with Blobly's brand
  identity: a morphing blob mark, pink-to-violet gradient, dark theme,
  and quick-question chips for the core FAQ.
- A **backend** Express server that receives each message, attaches
  Blobly's full knowledge base as a system prompt, forwards the
  conversation to the Gemini API, and returns the reply. The API key is
  kept server-side and never exposed to the browser.

The knowledge base covers exactly the questions Blobly needs answered
(company, mission, vision, products, internships, offer letter
verification, contact) and can be extended with more Q&A at any time by
editing one block of text — no code changes required.

## Technologies Used

| Layer         | Technology                                   |
|---------------|-----------------------------------------------|
| Frontend      | HTML5, CSS3, vanilla JavaScript (no framework) |
| Backend       | Node.js, Express                              |
| AI Model      | Google Gemini API (`generateContent`)         |
| Config        | dotenv (`.env` for API keys/settings)         |
| Fonts         | Space Grotesk (display), Inter (body)         |

No build step, bundler, or frontend framework is required — the frontend
is served as static files directly by Express.

## Installation Steps

**Prerequisites:** [Node.js](https://nodejs.org) v18 or newer, and a
Google Gemini API key from [Google AI Studio](https://aistudio.google.com/app/apikey).

1. **Unzip / clone the project** and open a terminal in its root folder
   (the folder containing `server.js` and `package.json`).

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Create your environment file**
   ```bash
   cp .env.example .env
   ```

4. **Add your Gemini API key**
   Open `.env` and replace the placeholder:
   ```
   GEMINI_API_KEY=your_gemini_api_key_here
   GEMINI_MODEL=gemini-2.5-flash
   PORT=3000
   ```
   Only `GEMINI_API_KEY` is required — `GEMINI_MODEL` and `PORT` are
   optional and already have sensible defaults.

## How to Run the Project

Start the server:
```bash
npm start
```

You should see:
```
Blobly Assistant server running at http://localhost:3000
```

Then open your browser to:
```
http://localhost:3000
```

**For development** (auto-restarts the server when files change):
```bash
npm run dev
```

**To stop the server:** press `Ctrl+C` in the terminal.

**Quick troubleshooting:**
- *"Server is missing GEMINI_API_KEY"* → `.env` wasn't created or the
  key wasn't pasted in. Redo steps 3–4 above and restart the server.
- *"I couldn't reach the server"* in the UI → the server isn't running,
  crashed, or you're browsing to the wrong port. Check the terminal for
  errors and confirm the port in your browser matches `.env`.
- *Gemini model errors* → set `GEMINI_MODEL` in `.env` to a model your
  API key currently has access to.

## Customizing the Knowledge Base

All of Blobly's company info lives in the `SYSTEM_PROMPT` constant near
the top of `server.js`. Edit it to update product details, add the real
Careers/internship/offer-letter links, add a real contact email, or add
more Q&A topics — the assistant answers strictly from this prompt, so
edits there are all that's needed.

## Notes

- Conversation history is kept in memory in the browser tab only (not
  persisted to a database) and the last 20 turns are sent back to Gemini
  for context on each request. Click **"New chat"** to reset.
- If `GEMINI_API_KEY` is missing, the server still starts and the UI
  still loads, but `/api/chat` returns a clear error until it's set.
